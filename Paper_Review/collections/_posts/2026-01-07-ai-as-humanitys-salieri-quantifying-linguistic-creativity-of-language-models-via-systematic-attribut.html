동까스LLM은 학습 과정에서 엄청 많은 인간 텍스트를 학습하기 때문에 의미 공간에서 인간 언어 분포의 중심에 가까운 출력을 내는 경향이 있다고 알고 있는데 이러한 LLM의 아키텍처적인 문제에서 어떻게 창의적인 답변을 내놓을 수 있을까라는 궁금점이 생김. + 다양한 분야의 지식을 알면 논문을 쓸 때 더 알 수 있겠다라는 생각이 듦3.8
마스킹테이프창의성을 평가하는 방식은 잘 모르겠다. 고전작가가 잘 나온 건 그냥 고전문학 스타일의 텍스트가 학습 데이터에 적게 분포하고 있어서 그런 건 아닐까? 라는 생각도 들었음. 예를 들어 판타지면 더 창의적인건가? 아이디어는 흥미롭지만, 아직은 부족한 연구라는 생각이 들었음. (논문을 잘 써서 커버했나 싶기도 함)3.5
귤'창의성을 측정한다'라는것 자체가 정말 애매하다고 생각을 해왔었는데, 기존에 얼마나 없었느냐~ 관점으로 바라보면 되겠구나. 근데 어떤 reference corpus를 기준으로 삼느냐에 따라 평가가 달라질 수 있다는 생각이 든다. 또 한편으로는 모델은 고정된 토큰 집합과 그 조합을 통해 문장을 생성하는데, 이것들을 얼마나 새롭게 조합하느냐의 문제로 볼 수 있지 않을까?4
수면장애 reference corpus와의 차이점으로 판단하는게 최선일까? 그 안에서 factuality가 손상될 수 있지 않을까? 물론 unrealistic한 생성을 해내는 것이 창의성의 척도인 건 맞지만, 단순히 fact만 뒤집어도 창의성이 높아질 수 있을까? (e.g. 이경호 교수님은 연대에서 학석박을 하셨다 ↔ 이경호 교수님은 경희대에서 학사를, 고대에서 석사를, MIT에서 박사를 하셨다) 3
이어폰창의성 평가 방법을 웹 텍스트 레퍼런스 기준으로 구현했다는 게, 납득 가면서도 되게 신박한 생각같다. LLM이 실제로 뜬금없이 웹 코퍼스 따라 말하는 경우도 많으니까 이런 오류 고치는 거랑도 관련지어 볼 수 있을 것 같고, 창의성 지표를 더 보완할 여지가 많아 보이지만 신박한 제안인 듯하다4
7일창의성 지표랑 hallucination score랑 발상적으로 비슷한 느낌이 들었다. 레퍼런스랑 동일할수록 정확도 관점에서는 오히려 좋은데… 이걸 함께 고려했으면 하는 아쉬움이 존재함.3.5
사과창의성 평가 방법을 Reference Corpus를 기준으로 삼는 것에 조금 의문이 들기는 함. 정확도와 Hallucination 문제를 해결하면서 창의성을 높이는 방법도 있는지..4

단순히 사람이 평가하기에는 모호하고 확장성이 떨어지니까 지표를 만들고, 잘 활용해보자

Background & Motivation

  • LLM이 창의력을 요하는 많은 직업을 대체하는 사례가 발생함
    • e.g. 웹툰, 영상, 작가 등등
    • 실제로 할리우드 주요 스튜디오들이 시나리오 작성 등의 제작과정에 LLM을 도입함

    ⇒ LLM은 그 어떤 사람보다 많은 작품을 접해왔기에(학습했기에), 새로운 차원의 창의성을 달성할 수 있지 않을까?

    ⇒ 과연 LLM이 사람의 창의성을 넘을 수 있을까?

  • 창의성은 정량화 및 비교가 어려움
    • 그동안 다양한 시도가 있었으나 (e.g. Torrance Test of Creative Thinking) 모두 human annotator에 의존함

      ⇒ 주관적이고, cost가 너무 큼

Contributions (What they’ve revealed)

  • 사람과 기계의 창의성에 관한 인사이트를 제공하기 위해, 창의성을 측정하는 지표인 CREATIVITY INDEX 제안
    • CREATIVITY INDEX란?
      • “web에 존재하는 human-written text를 얼마나 쉽게 재구성할 수 있는가”
        • 어떤 LLM이 생성한 텍스트가, 문맥이 긴데(n-gram의 n이 커짐) web에 거의 그대로 존재하는 텍스트라면, 그 LLM은 덜 창의적임
          • 만약 텍스트가 reference corpus에 그대로 존재한다면, CREATIVITY INDEX=0
        • 반대로, 문맥이 길어졌는데, web에서 찾을 수 없는 구간이 많은 텍스트를 생성한다면, 그 LLM은 더 창의적이다

        ** 측정할 때, reference corpus로 RedPajama를 사용함

        • why? 다양한 domain을 포함하는 large-scale web text corpus라서

      • 단순히 score가 아니라, 다양한 n의 길이에 대해 측정함
        • 텍스트길이/스타일에 영향을 덜 받도록 하기위함
    • professional writers 등 사람이 LLM보다 평균적으로 66.2% 높은 창의성을 가짐을 밝혀냄
      • 소설 발췌문, 현대시, 연설 녹취록 등 다양한 영역에서
      • verbatim 수준과 semantic 수준 모두에서
        • verbatim: 완전히 동일한 n-gram이 포함되는지
        • semantic: 의미적으로 매우 가까운 n-gram이 포함되는지
          • Word Mover’s Distance가 threshold 이하이면 semantic match로 판정
        • verbatim 수준만을 대상으로 했을 때(+52.2%) 보다, semantic 수준까지 함께 고려했을때 (+102.5%) 인간의 창의성이 LLM보다 높았음
    • RLHF가 LLM의 CREATIVITY INDEX를 평균 30.1% 감소시킴
      • 결과: -Base 는 RLFH 전

        ChatGPT, llama2-Chat, OLMO는 각각의 base가 RLHF된 버전

      • RLHF자체가, LLM이 인간의 선호대로 align되었을 테니까,,,
    • 다양한 인간 집단간의 CREATIVITY INDEX 비교
      • 허밍웨이, 디킨스 같은 고전문학작가는 가장 높은 수준의 창의성 보임
        • Details
          • Books published in 2023: BookMIA dataset
          • Classic Literature: 유명 작가의 고전 문학
          • Popular Teen Fiction: Goodreads’ book list로부터 가져온 추출한 인기 청소년 소설들
        • Insights
          • 창의성 뿐 아니라 작가의 문체, 집필시기 등의 요인을 받을 수도 있긴 함
          • 각 category 내에서도 창의성 지수 편차가 큼
            • e.g. Popular Teen Fiction 중 '헝거 게임'의 창의성 지수는 '트와일라잇'보다 35.4% 더 높음
    • 비공개 데이터로 학습된 GPT4의 경우, CREATIVITY INDEX를 우회적으로 측정하여 분석
      • 다른 LLM과 달리, GPT4는 Redpajama보다 최신/비공개 데이터로 학습되었기 때문
      • GPT-4와 유사한 knowledge update 시기인 공개모델(Gemma-7B, Llama3-8B, Mixtral-7B)이 생성한, ‘model-generated reference corpus’를 사용하여 평가

        (1) RedPajama 코퍼스에서 15만 개의 문장을 무작위로 추출

        (2) 공개모델들에게, 각 문장 뒤에 이어질 글을 문서 수준으로 생성하도록 함 

        Please generate a continuation for the following sentence: 
[PROMPT SENTENCE]

        (3) 이 텍스트를 reference corpus로 두고 CREATIVITY INDEX 측정

      • 인간의 평균 창의성 지수는 GPT-4보다 30.3% 더 높음
  • LLM 버전 업데이트에 활용하는, 웹상의 기존 text snippets의 사용 내역을 추적하는 알고리즘인 DJ SEARCH 제안
    • CREATIVITY INDEX 를 위해 reference corpus를 찾을 때, brute force로 하면 너무 비싸서 효율적으로 찾으려고!
      • 레퍼런스 corpus를 찾는게 DJ가 리믹스 할때 original composer에 credit하는 것과 유사하다고 생각해서 이렇게 지었다네요
    • 모든 (start position i, end position j) 쌍을 비교하기보다, 각 i에 대해 longest span을 찾자!
      • 이 과정에서 이미 계산한 정보를 재사용하자

      ⇒ 포인터를 움직이며 “이전보다 더 뒤에서 시작하거나/끝나는 n-gram만” 검사하도록 제한

      • verbatim matching여부를 먼저 판단하고, 이게 없으면 semantic matching 여부를 판단함
      • semantic matching 할 때, BM25로 x와 가장 유사한 텍스트들을 먼저 골라 그 subset에서만 WMD를 계산해 비용을 줄임
      • minimum n-gram length(k) = 5
      • WMD threshold = 0.95
