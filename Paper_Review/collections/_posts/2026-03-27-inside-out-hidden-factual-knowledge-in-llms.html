Summary

Motivation

  • LLM이 어떤 지식을 “안다”는 것은 무엇을 의미하는 걸까?
    • LLM이 출력으로 표현하는 것보다, 더 많은 지식을 알고 있지는 않을까?
      • 만약 그렇다면, LLM inference에 대해 개선해서 LLM의 퍼포먼스 올릴 수 있음
      • 또, inference 과정에서 어떤 지식이 왜 쓰이고 왜 쓰이지 않는지 이해할 수 있음

Contribution

  • LLM이 가지고 있는 Hidden Knowledge에 대해 정의하고 실험 프레임워크를 제안함
    • Definition: Existence of an internal function that ranks answers more accurately than any external function
    • 어떤 내부 함수(hidden state 활용)로 모든 외부 함수(토큰 생성 확률 활용)보다 답변을 더 잘 ranking할 수 있으면 내부 지식은 존재한다!
  • 논문에서 제안하는 지식의 정의는
    • 계산 가능함 → 실험적 연구에 유용
    • Closed-book QA 에서의 답변 정확도로 평가하지 않음 → 표면적이 아닌 내재적 지식에 집중
    • 외부적으로 표현하는 지식과 내부적으로 인코딩한 지식을 통일된 정의 하에서 측정함
  • Findings
    • Hidden knowledge는 실재함!
      • 이때 발견한 내부함수로 Closed-book QA 성능 개선함
    • 생각보다 깊게 숨겨져 있을 수 있고, 그러면 모델 생성에 전혀 반영이 안됨
      • 필자의 뇌피셜) Safety alignment가 되면 유해한 지식에 대해 알고 있어도 생성 안 함

Experiment Results

  • 내부지식은 존재한다!
    • 모델별로 내부지식을 잘 표현하는 정도는 다름
      • 라마는 잘하고 젬마는 잘 못함
    • 외부 함수 중에서는 P(True)가 제일 좋음

      → LLM은 서술형(생성)보다 TF문제(검증)에서 더 능숙함!

  • 1000번 생성해도 틀리는 문제에 대해서, 사실 알고 있을 수 있음
    • 1000번 샘플링 된것만 가지고 scoring 할 때 (회색) vs 1000번 샘플링 된거에 gold answer 주고 scoring할 때의 K∗K^*K∗﻿
    • 외부적으로 접근하면 gold 줘도 그것에 대해 맞는 사실이라고 인식 못함
    • 내부적으로 접근하면 gold 주면 그것에 대해 맞는 사실이라고 인식함
    • 내부적으로는 완벽히 알고 있는데 외부적으로 표현 못하는 질문이 전체의 7.2%정도 됨
  • Case Study
    • Q: Which company is Volvo B58 produced by?
      • 여기서 핵심은 볼보는 볼보로 보고, B58은 BMW의 엔진이 아니라 볼보의 버스임을 알아야 함
    • 외부적으로 접근하면 순위를 잘 못매기는데, 내부 프로빙으로 보면 순위를 완벽하게 잘 나눔
  • 큰 모델(32B)도 마찬가지일까?
    • 작은 규모로 32B 모델에서 해봤을 때, 마찬가지로 내부 프로빙에서 지식이 더 뚜렷함
  • Voting보다 내부 프로빙으로 답변 생성하면 더 잘함
    • Greedy가 생각보다 성능이 낮고, P(a∣q)P(a|q)P(a∣q)﻿가 greedy보다 나음
    • 내부 probing에서 가장 좋은 성능
    • 모델 스스로는 gold를 생성하지 못하더라도, gold answer를 주면, 내부 probing을 통해 그것이 정답이라고 모델은 말할 수 있음!
